Cài đặt lại thư mục node_modules

 Thư mục node_modules có dung lượng rất lớn so với toàn bộ dung lượng của source code. Mũ 2 source lên rồi nhân cho 100 nhiều khi còn không bằng
npm install

Thư mục sau khi thực hiện:

Chạy website

Sau khi tạo lại thư mục node_modules, có thể chạu ngay web lên mà không cần suy nghĩ

Build web dạng thường

ng serve --open
Trong đó:
  • --open: Tự động mở website sau khi build xong

Build web ở port khác (Mặc định port 4200)

ng serve --port 1234 --open

Build web theo ngôn ngữ

ng serve --configuration=production-vi --port 4200 --open
ng serve --configuration=production-en --port 4200 --open
Trong đó:
  • --configuration=production-vi: Build web với ngôn ngữ tiếng việt (Link web sẽ thành: http://localhost:4200/vi/)
  • --configuration=production-en: Build web với ngôn ngữ tiếng anh (Link web sẽ thành: http://localhost:4200/en/)

Thêm một trang mới

Các bước tạo module chức năng mới:

  • Tạo service
    Vị trí thư mục: /app/data/service
    ng g s data/service/abcd/abcd ng g s data/service/abcd/abcd
    Trong đó:
    • ● ng: Lệnh gọi Angular
    • ● g: generate
    • ● s: service
    • ● data/service/: Thư mục service
    • ● abcd (1): Tên thư mục chứa service
    • ● abcd (2): Tên service
  • Tạo module
    Vị trí thư mục: /app/modules
    ng g m modules/abcd --routing ng g m modules/abcd --routing
    Trong đó:
    • ● ng: Lệnh gọi Angular
    • ● g: generate
    • ● m: module
    • ● modules/: Thư mục modules
    • ● abcd: Tên module
    • ● --routing: Tuỳ chọn tạo route cho module (Tuỳ chọn này sẽ tạo ra 2 file: abcd.module.ts & abcd-routing.module.ts)

    Thư mục sau khi thực hiện:

  • Tạo component
    Vị trí thư mục: /app/modules
    ng g c modules/abcd ng g c modules/abcd
    Trong đó:
    • ● ng: Lệnh gọi Angular
    • ● g: generate
    • ● c: component
    • ● modules/: Thư mục modules
    • ● abcd: Tên module

    Thư mục sau khi thực hiện:

  • Edit file abcd.module.ts
  • Edit file abcd-routing.module.ts
  • Edit file app-routing.module.ts
    File app-routing.module.ts là file định nghĩa tất cả các route cho website.
    Sau khi thực hiện bước này, truy cập thử vào route đã chọn: http://localhost:4200/abcd để xem kết quả. Nếu website hiển thị: abcd works! thì cung xỉa, bạn đã hoàn thành!
  • Lập trình time
    Đến bước này thì khó có thể hướng dẫn tường tận nữa bởi mỗi module thì lại lập trình một cách khác nhau Google được sinh ra để làm tiếp nhiệm vụ thiên liêng và cao cả này
    Fighting!

Quy định Project

Cấu trúc thư mục
  • Mỗi route là một module riêng.
  • Các child route nằm trong thư mục 'pages' của module chính.
  • Các popup nằm trong thư mục 'dialogs' của module chính (cùng cấp với 'pages').
  • Ví dụ: Module procedure có các trang con như process, procost,...
    • Trang con process lại có các trang con process-add, process-diagram,...
  • Mỗi trang con như vậy đều có 2 file .module.ts & routing.module.ts
  • Các dialog không cần 2 file này!

Lấy thông tin cấu hình
  • Việc lấy thông tin cầu hình bắt buộc phải sử dụng EnvService, không import trực tiếp file environment.ts để lấy thông tin cấu hình.
  • Ngoài ra các thông tin cấu hình khác khi khai báo thêm cần phải đặt bên dưới trường config và luôn sử dụng Tiếng Anh.

Đa ngôn ngữ
  • Khi gọi API cần truyền thông tin languageTag của người dùng vào thuộc tính Accept-Language của Http Request Header.
      let headers = new HttpHeaders();
      headers = headers.set('Accept-Language', localStorage.getItem('language'));
    • Trong đó: localStorage.getItem('language') có thể thay thế bằng 'vi' hoặc 'en'

Đến đây thì là thời của anh TSLint:

  • B1: Dùng phím tắt Alt + Shift + F để giải quyết sơ bãi chiến trường.
  • B2: Tại các dòng còn báo lỗi bấm Ctr + . (Cần trô chấm) để hiển thị các tuỳ chọn fix.

SCSS

Hiện tại Digo đang dùng Router Angular để chuyển trang mà không load lại. Tốc độ khá nhanh tuy nhiên ở một số trường hợp CSS của trang trước không mất đi mà lại có tác dụng cho cả màn hình sau gây lỗi

Để tương tác với các phần tử Angular, đôi khi phải dùng thẻ ::ng-deep .Nhưng khi dùng thẻ này thì tất cả các class (hoặc id) trùng tên đều áp dụng css đó.

Ví dụ: Khi dùng ::ng-deep .test {color: #ff0000;} thì tất cả các thẻ có class test đều có màu chữ #ff0000

Vì thế khi viết css phải đi từ Class Cha Class Con Class Cháu
